# 棚番検索システム — 進捗メモ (Progress)

薬局の在庫の保管場所（棚番号）を検索する静的Webアプリ。**GitHub Pages で公開済み**。

- 公開URL: https://myuta7.github.io/PharmacyStockSearch/

最終更新: 2026-07-01（サジェスト方式へ画面刷新・ひらがな⇔カナ統一・UTF-8化・GitHub公開・名称修正まで完了）

## 1. 目的・概要

- 医薬品名を入力すると、CSV から一致する薬品を探して**棚番号**を表示するツール。
- 検索は**部分一致 + スペース区切りのAND検索**。
  - 例: `ｱﾑ 10 あす` → 3語すべてを含む `アムロジピンＯＤ錠１０ｍｇ「あすか」(E14)` がヒット。
- 文字種の違いは**正規化**で吸収して一致させる（実装済み・動作確認済み）。
  - Unicode **NFKC 正規化**（半角カナ⇔全角カナ、全角⇔半角数字、全角英字など）
  - **小文字化**（大小文字差を無視）
  - **カタカナ→ひらがな統一**（ひらがな⇔カタカナの差を吸収）
  - 例）`ｱﾑﾛ`・`アムロ`・`あむろ` はすべて同じ `あむろ` として一致。

## 2. 画面構成（実装済み・現行仕様）

**サジェスト（オートコンプリート）方式**。検索ボタンは廃止。

```
┌─ ヘッダー：「棚番検索システム」＋ 読み込みステータス ───────────┐
├───────────────────────────────────────────────────────────────┤
│  [ ヒント文 ]                                  [ すべてクリア ]  │
│  ───────────────────────────────────────────────────────────  │
│  行1  [ 入力欄（下にサジェスト候補が出る） ]   [ 棚番表示 ]      │
│  行2  [ 入力欄 ]                               [ 棚番表示 ]      │
│   …（全10行）…                                                 │
└───────────────────────────────────────────────────────────────┘
```

- 各行 ＝ **［医薬品名の入力欄］＋［棚番表示エリア（右）］** を横並び。全10行。
- 入力欄に文字を打つと、**その時点で一致する候補が入力欄の真下にリアルタイム表示**
  （最大30件、薬品名＋棚番を併記）。
- 候補を**クリック**（または ↑↓ で選び Enter）で確定 → 入力欄に薬品名が入り、
  **右の棚番エリアに棚番を表示**。
- Esc で候補を閉じる。フォーカスを外すと自動で閉じる。未選択時は棚番エリアは「―」。
- 上部に「すべてクリア」ボタン。

## 4. データ（CSV）仕様

- 文字コード:
  - `StockData_utf8.csv` … **UTF-8（BOMなし）** ← アプリはこちらを優先。
  - `StockData.csv` … **Shift-JIS**（`TextDecoder("shift-jis")` でデコード）。取得失敗時のフォールバック。
- 行数: 1625 行（ヘッダー1行 + データ）。
- 列構成: `薬品名, 内/外, 棚番1, 棚番2, 棚番3, 棚番4`
  - 1列目: 薬品名（全角カナ・全角数字を含む）
  - 2列目: 内 / 外（内服・外用など）※現状は検索・表示に未使用
  - 3〜6列目: 棚番号（空欄あり。複数の場合あり）
- 棚番号が空欄の薬品も存在する（表示上は「（棚番未登録）」）。

## 5. 検索ロジックの要点（script.js）

- `normalize(str)` = `str.normalize("NFKC").toLowerCase()`
  ＋ カタカナ(U+30A1〜U+30F6)を `charCode - 0x60` でひらがなへ変換。
  - 薬品名（読み込み時）と検索語（入力時）の**両方**に適用して比較。
- スペース（全半角）で区切った各語を、すべて含む（`Array.every` + `includes`）薬品を抽出。
- 各入力欄は**独立した1件の検索**として扱う（10行それぞれが個別の検索＋棚番表示）。
- サジェストは入力ごとに再計算し、最大 `MAX_SUGGEST`(=30) 件まで表示。

## 6. 動作確認済みの内容

- `ｱﾑﾛ 5 ｱｽ`（半角カナ）→ `アムロジピンＯＤ錠５ｍｇ「あすか」(E14)` にヒット
- `あむろ　５　あす`（ひらがな・全角空白）→ 同上ヒット
- `アムロ 5 アス`（全角カナ）→ 同上ヒット
- ※ ひらがな⇔カタカナをまたぐ検索も、正規化統一により一致するようになった。

## 7. ローカル確認方法

`file://` では `fetch()` がブロックされCSVを読めない。簡易サーバーで確認する。
このPCには Python も Node も無いため、**PowerShell製の `serve.ps1`** を用意済み。

プロジェクト直下で:

```
powershell -ExecutionPolicy Bypass -File .\serve.ps1
```

→ `Serving ... on http://localhost:8000/` と出たら、ブラウザで
`http://localhost:8000` を開く。停止は Ctrl+C。
