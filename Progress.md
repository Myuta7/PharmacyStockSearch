# 棚版検索システム — 進捗メモ (Progress)

薬局の在庫の保管場所（棚番号）を検索する静的Webアプリ。GitHub Pages で公開予定。

最終更新: 2026-07-01（仕様確定・UTF-8版CSV・スマホ最適化 対応）

---

## 1. 目的・概要

- 医薬品名を入力すると、CSV から一致する薬品を探して**棚番号**を表示するツール。
- 検索は**部分一致 + スペース区切りのAND検索**。
  - 例: `ｱﾑ 10 あす` → 3語すべてを含む `アムロジピンＯＤ錠１０ｍｇ「あすか」(E14)` がヒット。
- **半角カナ・全角カナ・全角/半角数字・ひらがな・大小文字**の違いは
  **Unicode NFKC 正規化 + 小文字化**で吸収して一致させる（実装済み・動作確認済み）。

## 2. 画面構成（実装済み）

```
┌─ ヘッダー：「棚版検索システム」＋ 読み込みステータス ──────────┐
├──────────────────────────────────────────────────────────────┤
│  [ 左：棚版表示エリア ]   [ 中央：入力欄×10（縦） ]  [ 右：検索ボタン ] │
│                                                    [ クリアボタン ]  │
└──────────────────────────────────────────────────────────────┘
```

- 左：検索結果（薬品名 + 棚番号）を表示。
- 中央：テキスト入力フィールド 10 個を縦に並べる。
- 右：検索ボタン（＋クリアボタンを追加済み）。Enter キーでも検索可。

## 4. データ（StockData.csv）仕様

- 文字コード: **Shift-JIS**（JS 側で `TextDecoder("shift-jis")` によりデコード）。
- 行数: 約 1626 行（ヘッダー1行 + データ）。
- 列構成: `薬品名, 内/外, 棚番1, 棚番2, 棚番3, 棚番4`
  - 1列目: 薬品名（全角カナ・全角数字を含む）
  - 2列目: 内 / 外（内服・外用など）
  - 3〜6列目: 棚番号（空欄あり。複数の場合あり）
- 棚番号が空欄の薬品も存在する（表示上は「（棚番未登録）」）。

## 5. 検索ロジックの要点（script.js）

- `normalize(str)` = `str.normalize("NFKC").toLowerCase()`
  - 薬品名（読み込み時）と検索語（検索時）の**両方**に適用して比較。
- スペース（全半角）で区切った各語を、すべて含む（`Array.every` + `includes`）薬品を抽出。
- 現状は 10 個の各入力欄を**独立した1件の検索**として扱い、
  入力済みの全欄をまとめて検索して左に結果を並べる設計。

## 6. 動作確認済みの内容

- `ｱﾑ 10`         → `アムロジピンＯＤ錠１０ｍｇ「あすか」(E14)`
- `ｱﾑ ｵｰﾊﾗ`      → オーハラのアムロジピン各種
- `アム １０ あす`（全角・ひらがな混在）→ 同上ヒット
- `ｾﾙｼﾝ`          → `２ｍｇセルシン錠(E19)`
- ※「ケミファ」はこのCSVに在庫が無いため0件（ロジックは正常）。

## 7. ローカル確認方法

`file://` では `fetch()` がブロックされCSVを読めない。簡易サーバーで確認する:

```
python -m http.server 8000
```

→ ブラウザで `http://localhost:8000` を開く。

## 8. GitHub Pages 公開手順

1. GitHub で新規リポジトリを作成。
2. `index.html / style.css / script.js / StockData.csv` をアップロード。
3. Settings → Pages で Branch = `main` (/root) を指定して保存。
4. 発行された URL で公開。
