# 棚番検索システム 仕様書

薬局の在庫（医薬品）の保管場所＝**棚番号**を、医薬品名の入力から素早く引き当てるための静的Webアプリケーション。

- 公開URL: https://myuta7.github.io/PharmacyStockSearch/
- リポジトリ: https://github.com/Myuta7/PharmacyStockSearch
- バージョン: 1.1（2026-07-02 時点／在庫データ 2026-07 版へ更新・複数棚番の全件表示・レイアウト改良）

---

## 1. 概要

| 項目 | 内容 |
|------|------|
| 目的 | 医薬品名（部分・略称・カナ/かな混在可）から棚番号を検索する |
| 形態 | サーバー不要の静的Webアプリ（HTML / CSS / JavaScript のみ） |
| データ | CSVファイル（在庫データ）をブラウザ側で読み込み |
| 動作環境 | モダンブラウザ（Chrome / Edge / Firefox / Safari の最新版） |
| 配信 | GitHub Pages |
| 想定利用者 | 薬局スタッフ（PC・スマホ両対応） |

### 特徴

- **サジェスト方式**：入力に応じて候補をリアルタイム表示し、クリックで確定。検索ボタン不要。
- **表記ゆれ吸収**：半角/全角・カタカナ/ひらがな・大小文字・全角/半角数字の差を無視して一致。
- **AND部分一致**：スペース区切りの語をすべて含む薬品に絞り込み。
- **10行同時**：最大10件の薬品を並行して調べられる。

### 2.3 操作フロー

1. 入力欄に医薬品名（の一部）を入力する。
2. 一致する候補が入力欄の直下にリアルタイム表示される。
3. 候補を **クリック**、または **↑/↓ キーで選択して Enter** で確定する。
4. 入力欄に薬品名が入り、その行の右に **棚番** が表示される。
5. 「すべてクリア」で初期状態に戻す。

### 2.4 キーボード操作

| キー | 動作 |
|------|------|
| ↓ / ↑ | 候補のハイライトを上下に移動 |
| Enter | ハイライト中の候補を確定 |
| Esc | 候補リストを閉じる |

### 2.5 レスポンシブ（スマホ対応）

- 画面幅 **641px 以上（PC）**：入力欄の**右**に棚番表示エリアを横並び。
- 画面幅 **640px 以下（スマホ）**：横並びをやめ、棚番表示エリアを**入力欄の下の行**へ回す
  （全幅を使えるので、複数棚番も折り返して見切れず表示できる）。
- 棚番が複数ある医薬品は、PC・スマホどちらでも `A / B / C` のように**全件**表示する
  （エリア内で折り返し）。

## 3. 検索仕様

### 3.1 正規化（表記ゆれの吸収）

比較の前に、薬品名（データ読込時）と検索語（入力時）の**両方**へ同じ正規化を適用する。

```js
function normalize(str) {
  return str
    .normalize("NFKC")                       // ① 互換文字を統一
    .toLowerCase()                           // ② 小文字化
    .replace(/[ァ-ヶ]/g, (ch) =>             // ③ カタカナ→ひらがな
      String.fromCharCode(ch.charCodeAt(0) - 0x60));
}
```

| ステップ | 効果 | 例 |
|----------|------|-----|
| ① NFKC | 半角カナ⇔全角カナ、全角⇔半角数字、全角英字→半角 | `ｱﾑ`→`アム`, `１０`→`10`, `ＭＧ`→`mg` |
| ② 小文字化 | 大小文字差を無視 | `MG`→`mg` |
| ③ カナ→かな | ひらがな⇔カタカナ差を吸収 | `アムロ`→`あむろ` |

→ `ｱﾑﾛ`・`アムロ`・`あむろ` はすべて `あむろ` に統一され、同じ薬品にヒットする。

### 3.2 マッチング

- 検索語をスペース（半角・全角どちらも）で分割し、**空でない各語をすべて含む**
  （AND・部分一致）薬品を抽出する。
- 判定は正規化後の文字列に対する `String.prototype.includes` による。

```
検索語 "ｱﾑﾛ 5 ｱｽ"
  → 正規化・分割: ["あむろ", "5", "あす"]
  → 薬品名 "アムロジピンＯＤ錠５ｍｇ「あすか」"（正規化: "あむろじぴん錠5mg「あすか」"）
     に3語すべて含まれる → ヒット
```

### 3.3 表示件数

- サジェスト候補は先頭から最大 **30 件**（`MAX_SUGGEST`）まで表示。
- 一致0件のときは「該当なし」を表示。

## 4. データ仕様

### 4.1 ファイル

| ファイル | 文字コード | 用途 |
|----------|-----------|------|
| `StockData_utf8.csv` | UTF-8（BOMなし） | **優先**して読み込む |
| `StockData.csv` | Shift-JIS | 上記の取得に失敗した場合のフォールバック |

- アプリはまず UTF-8 版を `fetch → text()` で読む。取得失敗時のみ Shift-JIS 版を
  `TextDecoder("shift-jis")` でデコードして読む。

### 4.2 CSV フォーマット

- 1行目はヘッダー（読み飛ばす）。
- 区切りはカンマ。**各フィールドはダブルクォート `"…"` で囲まれる**。
  フィールド内のカンマ（例 `舌下錠5,000JAU`）はクォート内では区切りとして扱わない。
  連続する `""` はエスケープされたダブルクォート 1 文字として解釈する。
- 行数: 1729 行（ヘッダー含む／データ 1728 件）。※2026-07 版

| 列 | 内容 | 使用 |
|----|------|------|
| 1 | 薬品名（全角カナ・全角数字を含む） | 検索・表示に使用 |
| 2〜8 | 棚番1〜棚番7（空欄あり・複数あり） | 空でないものを ` / ` 連結で表示 |

- 棚番がすべて空の薬品は「（棚番未登録）」と表示する。
- 旧版（〜2026-06）は 2 列目が「内/外」区分・棚番は 4 列・クォート無しだったが、
  2026-07 版で「内/外」列が廃止され、棚番が最大 7 列・全フィールドがクォート付きに変更された。

### 4.3 データ更新の注意

- 元データは Shift-JIS で受領する（例：`StockData2026_07.csv`）。更新時は
  **UTF-8（BOMなし）版 `StockData_utf8.csv` への再変換**と、Shift-JIS フォールバック
  `StockData.csv` の差し替えを行う。
  - 変換例（PowerShell。`$src` に受領した Shift-JIS ファイル名を指定）:
    ```powershell
    $src  = ".\StockData2026_07.csv"          # 受領した Shift-JIS 版
    $sjis = [System.Text.Encoding]::GetEncoding(932)
    $bytes = [System.IO.File]::ReadAllBytes($src)
    $text  = $sjis.GetString($bytes)
    # 1) アプリが優先読込する UTF-8（BOMなし）版
    [System.IO.File]::WriteAllText(".\StockData_utf8.csv", $text, (New-Object System.Text.UTF8Encoding($false)))
    # 2) Shift-JIS フォールバックも同じ新データへ同期
    [System.IO.File]::WriteAllBytes(".\StockData.csv", $bytes)
    ```
- CSV の列構成やクォート仕様が変わった場合は `script.js` の `parseCsv()` /
  `parseCsvLine()` の見直しも必要（2026-07 版で「内/外」列廃止・棚番7列・クォート付きへ対応済み）。

## 6. 動作環境・制約

- **サーバー不要**：完全にクライアントサイドで動作。バックエンドやDBは持たない。
- **ローカルでの `file://` 直開きは不可**：`fetch()` が制限されCSVを読めない。
  ローカル確認は `serve.ps1` 等の簡易サーバー経由で行う（本番はGitHub Pagesで問題なし）。
- データはページ読み込み時に全件メモリへ展開（1600件程度のため軽量）。
